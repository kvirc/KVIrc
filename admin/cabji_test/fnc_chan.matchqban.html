$chan.matchqban
Matches a mask against the channel ban list searching for +q bans (aka mute bans)
Usage
<string> $chan.matchqban([window_id],<complete_mask>)
Description
Warning: this function is network-specific, makes some (bad) assumptions about a non-standard channel mode and will probably be dropped in a future version.
Use $chan.matchmask instead.
Some networks use +q channel mode to set mute bans.
When a user mask matches such a ban, he won't be able to send messages to the channel.
The mute bans masks will be inserted in the normal channel bans list, with a percent sign % prepended.
This function returns the mute ban mask that matches <complete_mask> on channel identified by [window_id].
If no ban mask matches <complete_mask> an empty string is returned.
If [window_id] is empty, the current window is used.
If the window designated by [window_id] is not a channel a warning is printed and an empty string is returned.
This function is useful to determine if a mute ban set on the channel matches a user.
